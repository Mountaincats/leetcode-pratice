## 双指针
[TOC]

### 一、核心概念
双指针是一种通过操作**两个或多个指针**（索引）在数据结构中同向或反向移动，以协作完成遍历、搜索或计算的算法思想。它是滑动窗口的基础，区别在于双指针不一定局限于“窗口”这一形态，更侧重于**利用有序性或排除无效组合**来降低时间复杂度。

### 二、理论
**1. 指针移动模式**
- **快慢指针（Fast & Slow）**：两指针从同一起点出发，速度不同（或步长不同）
  - **应用**：检测环（链表）、寻找中点、删除倒数第 N 个节点、移除特定元素
- **对撞指针（Left & Right）**：指针分别位于序列两端，向中间逼近
  - **应用**：二分查找、两数之和（有序数组）、反转数组、回文校验。
- **分离指针（Multi-pointer）**：用于在两个或多个独立序列中遍历
  - **应用**：合并有序数组、寻找交集、归并排序

**2. 单调性利用**
- **有序数组**：对撞指针利用“和随指针移动单调递增/递减”的性质，排除不可能的解区间
- **链表**：快慢指针利用“速度差”计算长度差或相遇点

**3. 复杂度分析**
- **时间复杂度**：通常为 $O(n)$。虽然有两个指针，但每个指针通常只遍历数组一次（或常数次），避免了暴力法的 $O(n^2)$
- **空间复杂度**：$O(1)$，原地操作，无需额外存储

### 三、设计技巧
- **边界条件处理**：
  - 对撞指针：`while (left <= right)` 还是 `while (left < right)`？前者通常用于搜索（找到即停），后者用于收缩区间（需处理 `left==right` 的情况）
  - 快慢指针：注意 `fast->next->next` 可能导致空指针异常，需预判 `fast` 和 `fast->next` 的存在性
- **指针语义定义**：
  - 明确指针指向的是**元素**还是**空隙**（索引与边界的关系）
  - 在移除元素时，慢指针通常指向“已处理序列的末尾”，快指针指向“待检查元素”
- **虚拟头节点（Dummy Node）**：在使用快慢指针操作链表（如删除节点）时，引入虚拟头节点可以统一处理头节点被删除的边界情况，简化逻辑

### 四、示例模板

**1. 对撞指针（求两数之和）**
```c
int left = 0, right = n - 1;
while (left < right) {
    int sum = nums[left] + nums[right];
    if (sum == target) {
        return 1; // Found
    } else if (sum < target) {
        left++; // 和小了，左指针右移
    } else {
        right--; // 和大了，右指针左移
    }
}
```

**2. 快慢指针（删除有序数组重复项）**
```c
if (n == 0) return 0;
int slow = 0;
for (int fast = 1; fast < n; fast++) {
    if (nums[fast] != nums[slow]) {
        slow++;
        nums[slow] = nums[fast];
    }
}
return slow + 1; // 新长度
```

### 五、经典问题
| 指针类型 | 典型题目 | 核心逻辑 |
| :--- | :--- | :--- |
| **快慢指针** | 环形链表检测、快乐数 | 快指针每次走两步，慢指针走一步，相遇即有环 |
| **对撞指针** | 盛最多水的容器、三数之和 | 向中心逼近，根据状态更新极值 |
| **滑动窗口** | 最小覆盖子串、最长无重复子串 | 特殊的快慢指针，维护区间性质 |
| **分离双指针** | 合并两个有序数组、判断子序列 | 分别遍历两个数组，比较指针所指元素 |
