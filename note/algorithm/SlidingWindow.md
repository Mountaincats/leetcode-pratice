## 滑动窗口
[TOC]

### 一、核心概念
滑动窗口是一种在**线性数据结构**（如数组、字符串、链表）上，通过维护一个**左右边界（双指针）**的动态区间，来高效解决子区间问题的算法技巧。其核心思想是**窗口滑动，增量更新**，避免暴力枚举带来的高时间复杂度。

### 二、理论
**1. 窗口类型**
- **固定大小窗口**：窗口长度 $k$ 恒定。常用于计算“长度为 $k$ 的子数组的最大/平均值”、“出现频率”等
- **可变大小窗口**：窗口长度随约束条件动态变化。常用于求解“满足条件的最短/最长子数组”

**2. 指针移动策略**
- **右指针 (Right)**：负责**扩张**窗口，通常向右滑动以纳入新元素，直到满足特定条件或违反约束
- **左指针 (Left)**：负责**收缩**窗口，通常在窗口满足条件后向右移动，以尝试优化结果（如求最小值）或维持合法性

**3. 状态维护**
- **计数映射 (Counter)**：通常使用数组或哈希表（如 `uthash`）统计窗口内元素的频率、种类和特征
- **合法性判定**：定义一个函数或变量来判断当前窗口是否满足题目要求的约束条件（如“包含特定字符”、“和不超过目标值”）

**4. 适用场景特征**
- 求解**连续子数组/子串**的最值或存在性问题
- 问题具有**单调性**：即当右指针右移时，左指针无需回退（通常是求最值时）

### 三、设计技巧
- **区间边界定义**：明确窗口是**左闭右开** $[left, right)$ 还是**双闭** $[left, right]$。推荐使用**左闭右开**，此时窗口长度 $= right - left$，且当 $right$ 移动时，新增元素下标即为 $right$
- **收缩时机**：区分**求最短**与**求最长**。
  - 求最短（如最小覆盖子串）：窗口满足条件后，立即收缩左边界以寻找最优解
  - 求最长（如最长无重复子串）：窗口一旦违规，持续收缩直到合法为止
- **初始化与越界**：注意空窗口的处理。若使用 `while (left < right)` 且窗口为空时，`left == right`，需确保循环逻辑能正确处理初始状态

### 四、示例模板
```c
int left = 0, right = 0;
int ans = 0; // 或 INF
int window_state; // 例如：当前窗口的和、不同字符数等

while (right < n) {
    // 1. 右指针扩张：加入 nums[right]
    update_window_add(nums[right]);
    
    // 2. 左指针收缩：检查合法性
    while (window_is_invalid()) { // 或者 while (need_shrink())
        // 移除 nums[left]
        update_window_remove(nums[left]);
        left++;
    }
    
    // 3. 更新答案
    ans = max(ans, right - left + 1); // 或 min
    right++;
}
```

### 五、经典问题
| 问题类型 | 典型题目 | 窗口策略 |
| :--- | :--- | :--- |
| **固定窗口** | 滑动窗口最大值、子数组平均值 | 右移，维护单调队列或求和 |
| **可变窗口 (最长)** | 最长无重复子串、最长和为 K 的子数组 | 违规即收缩，达标不缩 |
| **可变窗口 (最短)** | 最小覆盖子串、和大于等于 K 的最短子数组 | 达标即收缩，违规不缩 |
