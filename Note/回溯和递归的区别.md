## 1. 核心关系
**回溯是一种特殊的递归应用**，或者说**回溯算法通常用递归来实现**。

可以把递归看作是一种“方法”，而回溯是使用这种方法解决“试探-回退”问题的“策略”。

## 2. 递归（Recursion）
**定义**：函数直接或间接调用自身的过程
**本质**：分治思想，将大问题分解为相似的子问题
**特点**：
- 有明确的递归终止条件
- 每次递归调用问题规模缩小
- 不需要“撤销”操作（纯递归）

**示例**：计算阶乘
```python
def factorial(n):
    if n <= 1:  # 终止条件
        return 1
    return n * factorial(n-1)  # 递归调用
```

## 3. 回溯（Backtracking）
**定义**：一种通过试探和回退寻找所有（或一个）解决方案的算法
**本质**：深度优先搜索 + 剪枝
**核心操作**：
1. 做出选择
2. 递归尝试
3. **撤销选择**（这是关键区别！）

**示例**：全排列问题
```python
def backtrack(nums, path, result):
    if len(path) == len(nums):  # 终止条件
        result.append(path[:])
        return
    
    for num in nums:
        if num in path:  # 剪枝
            continue
        path.append(num)    # 做出选择
        backtrack(nums, path, result)  # 递归尝试
        path.pop()         # 撤销选择！这是回溯的标志
```

## 4. 关键区别

| 方面 | 递归 | 回溯 |
|------|------|------|
| **目的** | 解决问题 | 寻找所有解 |
| **状态管理** | 通常不修改外部状态 | 需要维护和恢复状态 |
| **核心操作** | 函数调用自身 | 选择→递归→撤销 |
| **复杂度** | 通常O(n)或O(log n) | 通常指数级（如O(n!)） |
| **应用场景** | 计算类问题 | 组合、排列、搜索类问题 |

## 5. 关系总结

1. **包含关系**：回溯一定是递归（的实现方式），但递归不一定是回溯
2. **树形结构**：两者都可以用递归树理解
   - 递归：从根到叶的单一路径
   - 回溯：探索整个树，可能走多条路径
3. **状态管理**：
   - 递归：参数传递状态
   - 回溯：显式维护和恢复状态

## 6. 实例对比

**递归**（斐波那契数列）：
```python
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)  # 只是计算，不修改状态
```

**回溯**（N皇后问题）：
```python
def solve_n_queens(n):
    def backtrack(row, cols, diag1, diag2, board, result):
        if row == n:  # 找到解
            result.append(["".join(r) for r in board])
            return
        
        for col in range(n):
            if col in cols or (row-col) in diag1 or (row+col) in diag2:
                continue
            
            # 做出选择
            board[row][col] = 'Q'
            cols.add(col)
            diag1.add(row-col)
            diag2.add(row+col)
            
            # 递归尝试
            backtrack(row+1, cols, diag1, diag2, board, result)
            
            # 撤销选择（回溯关键！）
            board[row][col] = '.'
            cols.remove(col)
            diag1.remove(row-col)
            diag2.remove(row+col)
```

## 7. 记忆技巧
- 如果算法只是**分解问题+合并结果** → 是普通递归
- 如果算法需要**尝试多种可能+回退重试** → 是回溯
- 看到`path.pop()`、`visited[i] = False`这类“撤销”操作 → 就是回溯

**简单说**：递归是“一条路走到黑”，回溯是“多条路试试看，不行就回头”。
