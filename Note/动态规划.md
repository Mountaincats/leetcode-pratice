# 动态规划(Dynamic Programming)

## 一、基本概念

### 1. 核心思想
- 将复杂问题分解为**更小的子问题**
- **存储子问题解**以避免重复计算
- 通过**组合子问题解**得到原问题解

### 2. 适用条件
- **最优子结构**：问题的最优解包含子问题的最优解
- **重叠子问题**：不同的子问题会重复出现
- **无后效性**：未来决策只依赖当前状态，不依赖过去决策

## 二、实现方式

### 1. 自顶向下（Top-Down）记忆化递归
```python
# 斐波那契示例
memo = {}
def fib(n):
    if n <= 1: return n
    if n not in memo:
        memo[n] = fib(n-1) + fib(n-2)
    return memo[n]
```

### 2. 自底向上（Bottom-Up）递推
```python
# 斐波那契示例
def fib(n):
    if n <= 1: return n
    dp = [0] * (n+1)
    dp[0], dp[1] = 0, 1
    for i in range(2, n+1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]
```

## 三、解题步骤（五步法）

1. **定义状态**
   - 用数学符号表示子问题
   - 示例：`dp[i]` 表示到达第i步的最优解

2. **确定状态转移方程**
   - 找到状态之间的递推关系
   - 示例：`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`

3. **确定初始条件**
   - 确定最小子问题的解
   - 示例：`dp[0] = 0, dp[1] = 1`

4. **确定计算顺序**
   - 确保计算当前状态时，所需子状态已计算
   - 通常：从左到右，从小问题到大问题

5. **输出结果**
   - 从最终状态提取答案
   - 示例：`return dp[n]`

## 四、经典问题模板

### 1. 最长回文子串
```c
// 状态定义：dp[i][j] 表示 s[i..j] 是否是回文
// 状态转移：
if (s[i] == s[j]) {
    if (j - i < 3) dp[i][j] = true;
    else dp[i][j] = dp[i+1][j-1];
} else {
    dp[i][j] = false;
}
// 初始：dp[i][i] = true
```

### 2. 0-1背包问题
```c
// dp[i][w] 前i个物品，容量为w时的最大价值
for i in [1..n]:
    for w in [1..capacity]:
        if weight[i] > w:
            dp[i][w] = dp[i-1][w]
        else:
            dp[i][w] = max(dp[i-1][w], 
                          dp[i-1][w-weight[i]] + value[i])
```

### 3. 最长公共子序列
```c
if (text1[i-1] == text2[j-1]) {
    dp[i][j] = dp[i-1][j-1] + 1;
} else {
    dp[i][j] = max(dp[i-1][j], dp[i][j-1]);
}
```

## 五、空间优化技巧

### 1. 滚动数组
```c
// 二维降一维
int dp[2][W];  // 只保存前两行
// 或
int dp[W];  // 一维数组，倒序更新
```

### 2. 状态压缩
- 用位运算表示状态
- 适用于状态数较少的问题

## 六、常见问题类型

| 类型 | 特点 | 示例 |
|------|------|------|
| **线性DP** | 状态呈线性排列 | 最大子数组和、爬楼梯 |
| **区间DP** | 状态表示为区间 | 矩阵链乘、石子合并 |
| **树形DP** | 状态在树上 | 二叉树最大路径和 |
| **状态机DP** | 状态有转移关系 | 买卖股票、打家劫舍 |
| **数位DP** | 处理数字的数位 | 数字1的个数 |

## 七、分治 vs 动态规划

| 对比项 | 分治法 | 动态规划 |
|--------|--------|----------|
| **子问题** | 相互独立 | 相互重叠 |
| **存储** | 不存储子问题解 | 必须存储子问题解 |
| **时间** | 可能重复计算 | 避免重复计算 |
| **空间** | 通常较小 | 可能较大 |
| **适用** | 归并排序、快速排序 | 最优路径、序列匹配 |

**核心区别**：动态规划是"有记忆的分治法"

## 八、优化方法

### 1. 记忆化搜索
- 递归 + 缓存
- 适合状态转移复杂的情况

### 2. 状态压缩
- 减少空间复杂度
- 用位掩码表示状态

### 3. 斜率优化/四边形优化
- 高级优化技巧
- 优化特定形式的转移方程

## 九、注意事项

1. **边界条件**：仔细处理数组越界
2. **初始化**：确保所有状态都有初始值
3. **遍历顺序**：确保计算当前状态时，依赖的状态已计算
4. **空间优化**：先实现正确版本，再考虑优化
5. **调试技巧**：打印dp表查看状态变化

## 十、实战技巧

1. **从暴力法开始**，寻找重叠子问题
2. **画出状态转移图**，理清依赖关系
3. **先实现朴素版本**，再优化空间
4. **注意无后效性**，确保状态定义合理
5. **考虑初始化**，特别是边界情况

## 十一、复杂度总结

| 算法 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 暴力法 | 指数级 | O(1) |
| 记忆化搜索 | O(状态数) | O(状态数) |
| 自底向上DP | O(状态数×转移) | O(状态数) |
| 优化DP | O(状态数×转移) | O(优化后) |
